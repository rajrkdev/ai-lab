🔄 Step 4: Full RAG Pipeline

1
Load your documents

FAQ pages, help articles, product guides — any text content you want the system to know.

2
Embed documents → store in vector DB

Each document → 384-dim vector → stored in SQLite or similar. This is done once, offline.

3
User asks a question

The question is converted to a 384-dim vector using the same model — happens in milliseconds.

4
Find top matching documents

Cosine similarity: question vector vs all stored vectors. Top 3-5 highest scores = most relevant docs.

5
LLM generates the answer

Top matched docs + original question → sent to LLM (Claude etc.) → it generates a proper answer. Can be 100% local!
