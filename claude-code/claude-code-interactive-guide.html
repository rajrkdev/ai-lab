Architecture & Agentic Loop

Claude Code wraps Claude models in an agentic harness that provides tools, context management, and an execution environment. Every interaction follows a continuous agentic loop where Claude autonomously reasons, acts, and verifies until the task is complete.

The Agentic Loop — Core Execution Cycle
🔍
Gather Context
Search, read, grep, navigate dependencies
→
⚡
Take Action
Edit files, run commands, spawn subagents
→
✅
Verify Results
Run tests, check output, validate changes
↻

Phases blend together and repeat — Claude decides what each step requires based on previous results

🧠
Models that Reason
Opus 4.6 (80.8% SWE-bench), Sonnet 4.6 (79.6%), Haiku 4.5. Choose model by task complexity: Opus for architecture, Sonnet for daily work, Haiku for simple subagent tasks.
🔧
Tools that Act
Read, Write, Edit, MultiEdit, Bash, Grep, Glob, WebSearch, WebFetch, Agent, TodoWrite, LSP, NotebookEdit, Skill, MCP tools — full development toolkit.

CLAUDE.md Hierarchy — File Precedence

All levels are concatenated into context (not overriding). Files load in this priority order, with managed policy at the top having the highest authority.

CLAUDE.md Loading Order (All Concatenated)
P1
Managed Policy CLAUDE.md — Cannot be excluded
macOS: /Library/Application Support/ClaudeCode/CLAUDE.md
Linux: /etc/claude-code/CLAUDE.md
Windows: C:\Program Files\ClaudeCode\CLAUDE.md
P2
Project CLAUDE.md — Team-shared, version-controlled
./CLAUDE.md OR ./.claude/CLAUDE.md
P3
User CLAUDE.md — Personal, applies to all projects
~/.claude/CLAUDE.md
P4
Local CLAUDE.md — Personal, project-specific, gitignored
./CLAUDE.local.md

⚡ Claude walks UP the directory tree from CWD. Subdirectory files load on-demand when Claude reads files in those dirs.

Settings Cascade (Highest → Lowest Priority)

Settings Precedence — Deny Always Wins
1Managed settings — server-managed → MDM/OS-level → file-based

CLAUDE.md Best Practices

📏
Under 200 Lines
Keep concise. Use @imports and .claude/rules/ for splitting. Survives compaction, so every token counts.
🎯
Be Specific & Verifiable
"Use 2-space indentation in TypeScript" beats "Format code properly". Precision enables compliance checking.
📦
Use @Imports
@README for overview, @docs/arch.md for architecture. Recursive up to 5 hops. Keeps main file clean.
🏁
Start with /init
Generates a starter CLAUDE.md by analyzing your codebase. Then customize with your team's conventions.

Complete Tools Inventory

📖 Read NO PERM
Read text, images, PDFs (≤100 pages), Jupyter notebooks. Supports offset/limit for large files. Binary detection.
✏️ Write PERM REQ
Create or overwrite files. Auto-creates parent directories. Full content replacement.
🔧 Edit PERM REQ
Exact string search-and-replace. Optional replace_all flag. Surgical precision edits.
📝 MultiEdit PERM REQ
Multiple edits in a single atomic operation. Reduces permission prompts for batch changes.
🔍 Grep NO PERM
Regex search via ripgrep. Case-insensitive, context lines, file type filter, multiline patterns.
📂 Glob NO PERM
File pattern matching. ** recursive, {} alternatives. Results sorted by modification time.
💻 Bash PERM REQ
Execute commands in persistent shell. Configurable timeout. Background execution support.
🌐 WebSearch PERM REQ
Search the web for current information. Useful for docs, APIs, recent changes.
🔗 WebFetch PERM REQ
Fetch and analyze web pages. Domain-controllable via permission rules.
🤖 Agent NO PERM
Spawn subagents with custom type, model, prompt, background mode, and max turns.
📋 TodoWrite NO PERM
Manage task lists: id, content, status (pending/in_progress/completed), priority.
🧭 LSP NO PERM
Code intelligence: go-to-definition, find references, hover, symbols, call hierarchy.

Compaction Mechanics

⚡
Auto-Compact Trigger
Fires at ~83.5% capacity (~167K for 200K window). Configurable via CLAUDE_AUTOCOMPACT_PCT_OVERRIDE env var.
💾
What Survives
CLAUDE.md content is re-read from disk and re-injected fresh. Rules and loaded skills also persist.
📉
Degradation Point
Performance drops at ~147K-152K tokens (lost-in-the-middle attention), not at the 200K hard limit.
🚀
Speed
Since v2.0.64, compaction is effectively instant. /compact [focus] produces higher quality than auto.

2/clear between unrelated tasks; /compact at logical breakpoints
3Move specialized instructions from CLAUDE.md → skills (on-demand loading)
4/effort low for simple tasks; delegate verbose ops to subagents
5Be specific: "add validation to auth.ts" > "improve this codebase"
6Disable unused MCP servers; install LSP plugins to reduce file reads

Security & Permission Model

Permission Modes — Shift+Tab to Cycle
🟢Default — Prompts for file edits + all tool use
🔵Accept Edits — Auto-approves edits + common FS commands in CWD
🟡Plan — READ-ONLY: proposes but can't execute
🟣Auto — Classifier model decides safety
🟠Don't Ask — Denies anything not pre-approved
🔴Bypass — Auto-approves EVERYTHING ⚠️ Docker only!

Sandbox Mode

🍎
macOS — Seatbelt
OS-level filesystem and network sandbox. Enable with /sandbox. Reduces permission prompts by ~84%.
🐧
Linux — bubblewrap
Requires: sudo apt install bubblewrap socat. Filesystem writes restricted to CWD.
🛡️
Built-in Protections
curl/wget blocked. Command injection detection. Web fetch uses separate context (anti-prompt-injection).
