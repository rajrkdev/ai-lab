What is an Agentic Loop?

Section titled ŌĆ£What is an Agentic Loop?ŌĆØ

An agentic loop is a while loop where Claude keeps running until it decides itŌĆÖs done. Unlike a simple chat (one question ŌåÆ one answer), an agent can call tools, observe results, reason about them, call more tools, and keep going until the task is complete.

The key insight: Claude controls when the loop ends, not your code. Your code just checks a signal called stop_reason to know whether to continue or stop.

The Loop Lifecycle (Step by Step)

Section titled ŌĆ£The Loop Lifecycle (Step by Step)ŌĆØ

The Five Steps Explained

Section titled ŌĆ£The Five Steps ExplainedŌĆØ

Step 1: Send the prompt. Your code sends the userŌĆÖs message plus tool definitions to Claude. On subsequent iterations, you also include the full conversation history (all prior messages + tool results).

Step 2: Claude evaluates. Claude reads everything ŌĆö the system prompt, conversation history, tool definitions ŌĆö and decides what to do. It can produce text, call one or more tools, or both.

Step 3: Check stop_reason. The API response includes a stop_reason field:

  • "tool_use" ŌåÆ Claude wants to call tools. Continue the loop.
  • "end_turn" ŌåÆ Claude is finished. Stop the loop and return the text.
  • "max_tokens" ŌåÆ Claude hit the output limit. Response was truncated.
  • "refusal" ŌåÆ Claude declined due to safety concerns.
  • "pause_turn" ŌåÆ Server-side tool loop hit its iteration limit. Continue.
  • "stop_sequence" ŌåÆ Claude hit a custom stop sequence you defined.

Step 4: Execute tools. For each tool_use block in ClaudeŌĆÖs response, run the corresponding tool function and collect the result.

Step 5: Append to history. Add ClaudeŌĆÖs response as an assistant message, then add tool results as a user message with tool_result blocks. Each tool result must reference the tool_use_id from Step 4.

Model-Driven vs Pre-Configured Decision-Making

Section titled ŌĆ£Model-Driven vs Pre-Configured Decision-MakingŌĆØ

This is a critical exam distinction:

Model-Driven (Correct Agentic Pattern): Claude decides which tools to call and in what order based on the conversation context. You provide all available tools, and Claude reasons about which ones are needed.

Pre-Configured Pipeline (Not Truly Agentic): Your code dictates the sequence. Claude has no autonomy.

The exam tests this distinction because pre-configured pipelines canŌĆÖt adapt to unexpected situations (e.g., customer asks about something that doesnŌĆÖt require an order lookup).

Section titled ŌĆ£ŌÜĀ’ĖÅ Three Anti-Patterns (These ARE Wrong Answers)ŌĆØ

Anti-Pattern 1: Parsing Natural Language for Loop Termination

Why wrong: ClaudeŌĆÖs text is non-deterministic. It might phrase completion differently each time, or include ŌĆ£IŌĆÖm doneŌĆØ while still needing to call more tools. stop_reason is the ONLY reliable signal.

Anti-Pattern 2: Arbitrary Iteration Caps as Primary Mechanism

Why wrong: Either cuts off Claude too early (task incomplete) or runs too long (wasting tokens). max_turns as a SAFETY NET is fine; as the primary mechanism, itŌĆÖs wrong.

Why wrong: Claude often emits text ALONGSIDE tool calls, explaining what itŌĆÖs about to do. The presence of text doesnŌĆÖt mean Claude is done. Only stop_reason == "end_turn" means done.

Hub-and-Spoke Architecture

Section titled ŌĆ£Hub-and-Spoke ArchitectureŌĆØ

In a multi-agent system, a coordinator agent manages all inter-agent communication. Subagents never talk to each other directly ŌĆö everything flows through the coordinator.

Critical Rule: Subagents Have Isolated Context

Section titled ŌĆ£Critical Rule: Subagents Have Isolated ContextŌĆØ

Subagents do NOT automatically inherit the coordinatorŌĆÖs conversation history. They start fresh with only what you explicitly provide in their prompt.

Coordinator Responsibilities

Section titled ŌĆ£Coordinator ResponsibilitiesŌĆØ

The coordinator agent has four jobs:

  2. Delegation ŌĆö Decide which subagents to invoke (and which to skip)
  3. Result Aggregation ŌĆö Combine subagent outputs
  4. Quality Control ŌĆö Check if the combined output is complete
